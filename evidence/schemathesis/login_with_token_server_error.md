# Finding: Improper Error Handling in Token-Based Login Endpoint

## Summary

The Schemathesis fuzzing tool identified that the token-based login endpoint returns an Internal Server Error (HTTP 500) when receiving syntactically valid but invalid credentials. Manual validation confirmed that malformed requests are correctly rejected with HTTP 400, but specific invalid login attempts trigger a server-side error instead of a controlled authentication response.

## Endpoint

```http
POST /identity/api/auth/v2.7/user/login-with-token
```

## Discovery Method

Detected automatically during Schemathesis fuzzing and manually validated using curl.

## Reproduction Steps

### Request

```bash
curl -i -X POST http://localhost:8888/identity/api/auth/v2.7/user/login-with-token \
-H "Content-Type: application/json" \
-d '{"email":"vic@email.com","token":"000"}'
```

### Observed Response

```http
HTTP/1.1 500 Internal Server Error
Content-Type: application/json
```

```json
{
  "token": null,
  "type": "",
  "message": "Invalid Credentials",
  "mfaRequired": false
}
```

## Additional Validation

### Empty Fields

```bash
curl -i -X POST http://localhost:8888/identity/api/auth/v2.7/user/login-with-token \
-H "Content-Type: application/json" \
-d '{"email":"","token":""}'
```

Response:

```http
HTTP/1.1 400 Bad Request
```

### Missing Fields

```bash
curl -i -X POST http://localhost:8888/identity/api/auth/v2.7/user/login-with-token \
-H "Content-Type: application/json" \
-d '{}'
```

Response:

```http
HTTP/1.1 400 Bad Request
```

These tests indicate that input validation is implemented correctly for malformed requests. However, the endpoint still returns an Internal Server Error when processing invalid credentials that satisfy the expected input format.

## Expected Behavior

Invalid credentials should result in a controlled authentication-related response such as:

```http
401 Unauthorized
```

or

```http
400 Bad Request
```

rather than an Internal Server Error.

## Security Impact

The endpoint exposes a server-side error condition during a normal authentication failure scenario.

Although no authentication bypass was identified, the behavior indicates improper exception handling in the token-based authentication workflow. Internal server errors generated by invalid credentials may reveal implementation weaknesses and reduce application robustness.
